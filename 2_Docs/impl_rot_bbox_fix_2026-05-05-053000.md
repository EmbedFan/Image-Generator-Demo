## 1. Root Cause

### 1.1 Change that introduced the regression

FEA-007 changed the frame-body parse call from:

```python
# Before FEA-007
_parse_drawing_commands("end_frame", in_func_body=False, ...)
```

to:

```python
# After FEA-007
_parse_drawing_commands("end_frame", in_func_body=True, allow_var_stmts=True)
```

When `in_func_body=True`, `_parse_value` wraps every bare NUMBER token as
`ExprFactor(LengthValue(number, ''))` instead of returning a plain `LengthValue`.

This means that `radius=60` in a frame-level primitive is now stored in the AST as
`ExprFactor(LengthValue(60, ''))` — not as `LengthValue(60, '')`.

### 1.2 Why primitive rendering still works

`PrimitiveDispatcher._dispatch_one` (lines 138–154 of
`imagegen/rendering/primitive_dispatcher.py`) calls `self._resolve_cmd(cmd, variable_store)`
before handing the command to a renderer. `_resolve_cmd._resolve_value` unwraps
`ExprFactor(LengthValue(...))` back to `LengthValue(...)`. Primitive renderers
therefore receive correctly-typed params.

### 1.3 Why bbox overlays break

`frame_runner.py` calls:

```python
render_bbox_overlays(canvas, geometry_cmds + text_cmds, symbol_table)
```

These lists hold the **original, unresolved** AST nodes straight from the parser.
`bbox_renderer._compute_aabb` does checks like:

```python
r_val = params.get("radius")
if isinstance(center, PointValue) and isinstance(r_val, LengthValue):  # ← FAILS
```

`r_val` is now `ExprFactor(LengthValue(60, ''))`, not `LengthValue`. The `isinstance`
guard fails → `_compute_aabb` returns `None` → no overlay is drawn.

Affected primitive types: `CircleNode` (line 122), `PieNode`/`ArcNode` (lines 132–136),
`SquareNode` (lines 145–146), `FontNode` (line 199), `ImagePrimNode` (lines 213–214).
Also `_object_inst_aabb` for `ObjectInst` scale/rotate AABB expansion (lines 249, 260).

### 1.4 Why object rotate and scale break

`PrimitiveDispatcher._dispatch_one` dispatches `ObjectInst` as:

```python
if isinstance(cmd, ObjectInst):
    self._object_instantiator.instantiate(
        canvas, cmd, symbol_table, self, base_dir, frame_index
    )
    return
```

The **original, unresolved** `cmd` is passed directly — `_resolve_cmd` is never called for
`ObjectInst`. Inside `object_instantiator.instantiate`:

- **Line 68:** `isinstance(node.params["width"], LengthValue)` → `False` if width was
  supplied at the call site, because it is now `ExprFactor(LengthValue(...))`.
- **Line 72:** `isinstance(scale_param, LengthValue)` → `False` for `scale=2.25`
  → `factor` never computed → scale silently ignored; object renders at native size.
- **Line 120:** `isinstance(rotate_param, LengthValue)` → `False` for `rotate=35`
  → rotation block skipped → object renders without rotation.

---

## 2. Fix Strategy

Add a tiny `_unwrap_length(val)` helper to the two affected files and apply it at every
point where an `isinstance(x, LengthValue)` check currently fails on wrapped values.
No changes to the parser, AST, or VariableStore are required.

The helper unwraps one layer of `ExprFactor` when its payload is a `LengthValue`:

```python
def _unwrap_length(val: object) -> LengthValue | None:
    if isinstance(val, LengthValue):
        return val
    if isinstance(val, ExprFactor) and isinstance(val.value, LengthValue):
        return val.value
    return None
```

This is backward-compatible: frames that were parsed with `in_func_body=False`
(object bodies) still produce plain `LengthValue` values, which the helper accepts unchanged.

## 5. Scope — What Is NOT Changed

| File | Reason unchanged |
|---|---|
| `imagegen/parser.py` | Root cause is at parse time but the fix is applied at dispatch/rendering; changing parsing semantics would be a larger, higher-risk change |
| `imagegen/rendering/primitive_dispatcher.py` | `_resolve_cmd` already correctly unwraps `ExprFactor` for primitive nodes |
| `imagegen/rendering/transform_applier.py` | Receives already-resolved params from primitive renderers via `_dispatch_one → _resolve_cmd → renderer.render`; no unwrapping needed |
| `imagegen/frame_runner.py` | Fix is self-contained in the two rendering files; no changes to how commands are collected or ordered |
| `imagegen/rendering/function_executor.py` | Function bodies already used `in_func_body=True` before FEA-007 and function-executor code is unaffected |

## 7. Risk Assessment

| Risk | Likelihood | Mitigation |
|---|---|---|
| `_unwrap_length` misidentifies a non-numeric ExprFactor as a length | Low | Helper requires `isinstance(val.value, LengthValue)` — only numeric literals wrapped by the parser match |
| Other callers of the same params fields not covered | Low | `TransformApplier` receives already-resolved params; semantic validator runs before rendering; no other consumers identified |
| Template-attribute `LengthValue` (object body) breaks | None | Template attributes are parsed with `in_func_body=False` → plain `LengthValue` → helper returns it as-is |
